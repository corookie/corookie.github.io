前端笔记总结

JS方面

1. this 的指向

  • this默认值为window
  • 在函数内默认也是windows

  • 但是，当函数被作为对象的方法时，this指向该对象。但是，当被嵌套函数调用时，this指向windows

  • 闭包中默认指向windows

  • 当函数作为对象的方法时调用丢失了自己绑定的对象，又会默认指向windows

  • 当使用new 关键字的时候，内部的this 指向该实例。

3.异步(promise):angry:

  • 什么是异步

我们都知道js是单线程的，但再实际生活中，肯定不能因为一个任务就不执行后续任务，因此在实际中js是异步执行的。那这样的话问题来了，假设遇到这种情况怎么办？(简单来说，异步就是先执行，再调用)

这就是异步编程的不方便之处，我们希望sentimeout执行完再执行，end，那么怎么解决呢？

老方法使用的回调函数，因为函数是一等公民，所以可以作为参数使用。

回调函数同样存在问题，假设callback还需要调用另一个函数，这就会导致回调地狱的产生。无限嵌套。因此新时代的前端人使用promise 代替回调函数:smiley_cat:，下面且听我细细道来。

  • promise :upside_down_face:

期约是个对象，代表异步操作的返回值或者结果值。有三种状态，待定，解决或者拒绝，只有解决了才会运行then，调用回调函数。

有几个名词，期约解决: 当把回调c传递给then(),方法时，会返回一个期约p。当c的结果是个值，p 就会得到解决并以该值兑现，当c的结果是个期约V的话，这时候我们会说，我们说p会得到解决，但并没有兑现，想兑现则需要V落定后才能兑现，()

  • catch 和finally

catch一般放在末尾,用于处理各类错误，instanceof 或者 typeError。

finally 的回调会在期约落地后被调用，不管期约状态与否，适合完成那种清理代码类的工作

  • 箭头函数
  • async 和 await

这两个可以让异步代码看起来像同步代码 十分的简洁。

await一般用法 let s = await promise() , 只有当await 后面的期约兑现或解决 ,才会执行后面的代码，因此await只允许在async函数内使用。

4.let和var

  • let和var在全局作用域声明时 二者类似

  • let声明的变量 不能作为全局变量的属性 也就是说windows里没有let声明的变量。

  • 在函数作用域中，二者一样，但是在块作用域中，例如for循环中let声明的只在循环中有效，而var则都有效。

    因此，let在块作用域中作用更大

6.文档对象模型和浏览器程序的执行

​ 客服端编程中最重要的对象就是Document对象，它代表浏览器窗口和标签页中显示的html文档，dom(document object model)可以操作html文档。 每个浏览器或者标签页都有一个全局对象，全局对象有个属性叫windows 他的值等于全局对象本身。所以可以访问windows来引用全局对象。给全局对象添加属性能够全局引用(var 或者 function声明),es6中的const let class声明不会给全局对象增加属性，但是会被定义在一个共享的命名空间，因此也能够被引用。 然而在模块中，顶级(即位于函数或者类之外的) 的常量或者变量、函数和类都是模块私有的，可以被导出。

JS程序的执行

  • 浏览器创建document对象并解析整个html文档 document.readstate = loading
  • 当遇见无(async,defer,type = module)的脚本会把标签添加到文档，并执行。此期间html暂停解析
  • 当遇见async 的脚本时，此时会先下载脚本，会尽快执行。此时html持续解析。3
  • 文档解析完成后，会 d.readstate = interactive
  • defer 和不带async属性的脚本会按照顺序执行。异步脚本也可能此时执行。
  • 浏览器在d对象上发放DOMcontentLoaded”事件，程序执行进入到异步事件驱动状态
  • 此时仍然可能有异步脚本未执行。
  • 当异步脚本和内容全部加载出来时(例如较大图片)，d.state = load ,浏览器在d对象上派发load事件，从这一刻起，作为对用户输入事件，网络事件，定时器超时等的响应，浏览器开始异步调用事件处理程序。

7.事件

事件就是浏览器会在文档、浏览器或者某些元素或与之关联的对象发生某些值得关注的事时生成事件。 例如 浏览器加载完文档，用户点击某个超链接，也会在用户敲击键盘时生成事件。

事件类型有很多，表示发生事件的类型，例如 mousemove,表示鼠标发生移动。

事件目标是个对象，表示事件发生在该对象上或者于该对象有关，常用的对象有 element windows document。

事件处理程序和事件监听器 是个函数，负责处理响应事件，常用 addeventlistener。

事件对象 是与特定事件关联的对象，包含有关该事件的细节，并作为事件处理程序的参数传入。所有事件对象都包括两个参数 type和 target ,前者代表 事件类型 后者代表事件目标。
