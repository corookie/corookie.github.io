vue中diff算法实现

​ vue中为了在拥有可维护性情况下，实现一定的性能优化，采取了虚拟dom的方式对dom进行更新，虚拟dom就是一个对象，里面是树状结构，每个节点包含着一个标签。当一系列对dom的操作结束后，vue生成更新后的虚拟dom，然后对比更新前的虚拟dom，找出更新的地方生成真实dom渲染到页面，以下是vue的更新过程。

  • 创建新的虚拟 DOM 树：当组件的状态发生改变时，Vue 会创建一个新的虚拟 DOM 树。
  • 对比新旧虚拟 DOM 树的差异：Vue 会对比新旧两个虚拟 DOM 树的差异，这个过程称为 diff。Vue diff 算法的核心思想是只更新发生变化的部分，而不是整个 DOM。
  • 更新视图：根据差异，Vue 会生成一系列操作指令，然后将这些操作指令应用到真实 DOM 上，以更新视图。

​ 为了能够更高效率的找出修改的dom，vue有一系列的策略，这些策略被成为diff算法，下面是diff算法的一些流程，diff算法有个原则，就是只会进行同层级的节点对比，不会跨层对比。

  1. 如果新旧两个节点的 tag 不同，直接替换节点。
  2. 如果新旧两个节点的 key 不同，则认为它们是不同的节点，直接替换节点。
  3. 如果新旧两个节点的 tag 和 key 都相同，则对比它们的属性和子节点。
  4. 如果新旧两个节点的属性不同，则更新属性。
  5. 如果新旧两个节点的子节点不同，则对子节点进行递归比较和更新。

根据比较规则，diff 算法可以高效地对比和更新虚拟 DOM 树，从而实现对视图的快速更新。
