http请求

发表于2023-04-01|更新于2023-04-01|AJAX
|阅读量:

http

​ 在面试的时候发现对http相关的知识掌握还是不太牢固，因此做个总结，本文大概分为 http请求(请求方式，请求头，请求体)，http响应(响应头的设置[缓存等])，http协议(1.0,1.1,2.0)

  • http请求

    http请求是客户端和服务端通信的一种方式，当用户需要从服务器获取数据时，会使用http协议向服务器发起请求。

    • 请求方式

      • get:简单高效，用于获取服务器上的资源，通常会把参数放在url后面,多个参数用&连接(因此参数长度较小，一般不超过2kb)，且返回结果会被浏览器缓存，因此适合获取不变的资源，没有请求体。

      • post，适用于提交数据，参数放在请求体中，不暴露在url上(参数)，结果不会被浏览器缓存。

      • put：用于更新服务器上的资源，需要指定要更新资源的完整URL，因此可以对服务器上的资源进行更新、创建和替换操作。

      • delete：用于删除服务器资源，需要指定完整的URL。

    • 请求头设置

      ​ 请求头会提供一些请求的信息，请求方法，主机地址，用户代理等，下面是一些常用的请求头字段

      • Accept:指定客户端能够接收的数据类型，例如’text/html’、’img/jpg’

      • Accept-Encoding:指定客户端能够接受的编码方式，例如’gzip’、’deflate’

      • Authorization:身份验证的凭证，例如 Basic 认证的用户名和密码等。

      • cache-control:指定缓存机制，例如:’no-cache(不走强缓存)’ 、no-store(不走协商缓存)、’max - age’ = 3600等

      • connection:指定连接类型，例如’’keep-alive’表示保持长连接。(http1.1以上默认开启)

      • Content-Length：指定请求体的长度，单位为字节。

      • Content-Type：指定请求体的类型，例如 application/json、application/x-www-form-urlencoded 等。

      • Cookie：用于在客户端和服务端之间传递 cookie 信息。

      • Host：指定请求的主机名和端口号

      ​ 除了上述常见的HTTP请求头字段之外，还有一些其他的字段，例如 If-Modified-Since、If-None-Match、Range 等。这些字段的具体含义和使用方法可以根据不同的需求来了解和使用。

    • 响应头设置

      • content-type:响应类型
      • content-length：指定响应体的长度(以字节为单位)
      • location:重定向的url，用于将客户重定向到其他url
      • set-cookie:指定要在客户端上设置的cookie
      • cache-control:响应的缓存策略，no-cache ，优先级大于expires
      • expires：指定响应过期时间，客户端可以使用这个信息决定是否应该使用缓存的响应，这个和请求头的max-age相关，但格式不一样，这是个字符串格式的时间。

    • 浏览器缓存策略

      ​ 当用户第一次访问某些域名时，会把一些响应结果和字段保存在浏览器缓存中(保存在本地硬盘或者内存中)，当再次向该域名发起请求时，会先查看浏览器缓存，对比其中的cache-control(优先)中的max-age字段或者Expires字段是否过期。未过期则直接读取。(强制缓存)

      ​ 当资源过期或者需要刷新缓存的时候(f5刷新页面),会触发协商缓存，有些http响应头中会添加(Last-Modified(时间戳，以s为单位)和ETag字段(内容标识，更准确但耗性能，因此优先级高))，这二者可以检测资源是否发生变化，用来进行缓存验证。

      ​ 浏览器向服务器发送缓存验证请求(If-Modified-Since(last-modified)、If-None-Match(Etag))，假设服务器对比后发现资源未变化，则会返回状态码304，并且携带新的缓存过期时间；假设资源发生变化则会返回新的资源。

  • http协议

    • http1.1

      • 默认启用持久链接请求头加入connection字段，默认保持keep-alive
      • 管道化请求，允许在同一个tcp连接发送多个请求，然后服务器按照响应顺序返回。
      • 身份认证，响应头的authorization
      • 缓存控制，cache-control

    • http2.0

      • 头部压缩，把多个请求的头部字段压缩为一个表，然后在具体请求头用索引取对应的值。
      • 多路复用，允许同一个tcp连接发送多个请求，并且不用按照顺序返回，请求和响应交错传输。
      • 引入帧和流：HTTP2 将 HTTP 报文分割成多个二进制帧，从而允许多个请求和响应交错传输，从而避免了 HTTP/1.x 的串行阻塞问题，提高了网络传输效率。
      • 服务器推送：
      • 优先级（Priority）：HTTP2 允许客户端和服务器设置请求和响应的优先级，从而控制网络带宽的分配和利用，提高了网络传输的效率和公平性。
